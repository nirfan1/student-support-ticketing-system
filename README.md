# Student Support Ticketing System

## Project Overview

Student Support Ticketing System is a responsive web prototype for managing university student support requests. It helps students submit issues and helps staff organise requests by category, priority, and status.

This project is suitable for an Information Systems, IT Business Analyst, Systems Analyst, IT Support Analyst, or Digital Transformation portfolio.

## Problem Statement

Student support issues can become confusing when requests are handled through scattered emails, phone calls, screenshots, and separate forms. Students need a clear way to submit an issue, and staff need a simple workflow to review, prioritise, update, and resolve requests.

## Main Features

- Student support request form
- Categories for enrolment issues, rule waivers, degree progression, prerequisite problems, and technical access
- Priority selection: High, Medium, Low
- Admin ticket queue
- Ticket status updates: Open, In Progress, Resolved
- Dashboard metrics for total, open, in-progress, and resolved tickets
- Status and category filters
- Sample ticket export
- Responsive design for desktop and mobile

## How to Run

Open `index.html` in any browser.

No installation is required.

## How to Publish with GitHub Pages

1. Upload this project to a public GitHub repository.
2. Go to repository **Settings**.
3. Open **Pages**.
4. Under **Build and deployment**, select:
   - Source: Deploy from a branch
   - Branch: main
   - Folder: /root
5. Click **Save**.
